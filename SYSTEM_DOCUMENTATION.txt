1. SYSTEM OVERVIEW
================================================================================

HU-SIMS is a comprehensive school information management system designed for
Haramaya University Non-Boarding Secondary School. The system enables efficient
management of students, teachers, parents, academic records, attendance,
reports, timetables, and communication.

LIVE DEMO:
- Frontend: https://hunb-sims.vercel.app
- Backend: https://hu-sims-backend.onrender.com

SYSTEM PURPOSE:
- Centralized student and staff management
- Academic record tracking and reporting
- Attendance monitoring and absence alerts
- Timetable management and scheduling
- Internal communication and announcements
- Role-based access control for different user types
- System administration and monitoring

================================================================================
2. ARCHITECTURE
================================================================================

The system follows a client-server architecture with separate frontend and
backend applications:

FRONTEND ARCHITECTURE:
- React 19 with TypeScript for type safety
- Component-based architecture with reusable UI components
- State management using Zustand
- Data fetching with TanStack Query (React Query)
- Routing with React Router DOM
- Form handling with React Hook Form and Zod validation

BACKEND ARCHITECTURE:
- RESTful API built with Express.js
- MongoDB for data persistence (MongoDB Atlas)
- JWT-based authentication
- Role-based access control (RBAC)
- Middleware for authentication, authorization, and error handling
- Modular controller-route-model structure

COMMUNICATION:
- HTTP/REST API between frontend and backend
- JWT tokens for authentication
- CORS enabled for cross-origin requests

4. USER ROLES AND PERMISSIONS
================================================================================

The system implements a role-based access control (RBAC) system with 5 roles:

1. SYSTEM ADMIN (SystemAdmin)
   Purpose: Technical system management, user accounts, security, backups
   Permissions:
   - User account creation and role assignment (full access)
   - Student account management (READ, WRITE, EDIT only)
   - Teacher account management (READ, WRITE, EDIT only)
   - Parent account management (READ, WRITE, EDIT only)
   - Messages (READ, WRITE)
   - System settings (full access)
   - EXCLUDED: Academic records, attendance, timetables, reports, certificates

2. SCHOOL ADMIN (SchoolAdmin)
   Purpose: School operations, academic administration, day-to-day management
   Permissions:
   - Users (READ, WRITE, EDIT, DELETE)
   - Students (full access)
   - Teachers (READ, WRITE, EDIT)
   - Parents (full access)
   - Academic records (full access)
   - Attendance (full access)
   - Timetables (full access)
   - Reports (READ, WRITE, EDIT, DELETE)
   - Messages (READ, WRITE)
   - Certificates (READ, WRITE, APPROVE)
   - Settings (READ, EDIT)

3. TEACHER (Teacher)
   Purpose: Classroom operations, grading, attendance, student communication
   Permissions:
   - Users (READ only)
   - Students (READ only)
   - Teachers (READ only)
   - Parents (READ only)
   - Academic records (READ, WRITE, EDIT)
   - Attendance (READ, WRITE, EDIT)
   - Timetables (READ only)
   - Reports (READ, WRITE)
   - Messages (READ, WRITE)
   - Certificates (READ only)

4. STUDENT (Student)
   Purpose: Access to personal academic information and school communication
   Permissions:
   - Students (READ only - own records)
   - Teachers (READ only)
   - Academic records (READ only - own records)
   - Attendance (READ only - own records)
   - Timetables (READ only)
   - Reports (READ only - own reports)
   - Messages (READ, WRITE)
   - Certificates (READ only)

5. PARENT (Parent)
   Purpose: Access to linked child records, progress, and school communication
   Permissions:
   - Students (READ only - linked children)
   - Teachers (READ only)
   - Parents (READ only)
   - Academic records (READ only - linked children)
   - Attendance (READ only - linked children)
   - Timetables (READ only)
   - Reports (READ only - linked children)
   - Messages (READ, WRITE)
   - Certificates (READ only - linked children)

PERMISSION TYPES:
- READ: View data
- WRITE: Create new data
- EDIT: Modify existing data
- DELETE: Remove data
- APPROVE: Approve/reject submissions
- MANAGE: Full administrative control
- ADMIN: Highest level permission

RESOURCE TYPES:
- users: User account management
- students: Student records
- teachers: Teacher records
- parents: Parent records
- academic-records: Grades and academic performance
- attendance: Attendance tracking
- timetables: Class schedules
- reports: Performance reports
- messages: Internal communication
- certificates: Certificate generation
- settings: System configuration

6. BACKEND API ENDPOINTS
================================================================================

The HU-SIMS backend has **201 total API endpoints** organized into 15 modules:

Authentication (/api/auth) - 11 endpoints:
- POST /login - User login
- POST /verify-2fa - Two-factor authentication verification
- POST /logout - User logout
- POST /verify-email - Email verification and password setup
- POST /resend-verification - Resend verification email
- POST /forgot-password - Request password reset
- POST /reset-password - Reset password with token
- GET /me - Get current user profile
- PUT /me - Update current user profile
- PUT /appearance - Update appearance settings
- POST /change-password - Change password (authenticated)

Users (/api/users) - 2 endpoints:
- POST / - Create a new user (Admin only)
- GET / - Get all users (Admin only)

Students (/api/students) - 16 endpoints:
- GET /profile - Get student's own profile
- GET /grades - Get student's own grades
- GET /average - Calculate student's average marks
- GET /attendance - Get student's own attendance records
- GET /schedule - Get student's timetable/schedule
- GET /announcements - Get announcements for students
- GET /parents - Get student's linked parents
- GET /certificates - Get student's certificates
- GET /reports - Get student's reports
- GET /by-student-id/:studentId - Get student by custom ID (Admin/Teacher)
- POST / - Create student (Admin only)
- GET /:id/academic-documents/:documentId/download - Download academic document
- PUT /:id - Update student (Admin/Teacher)
- DELETE /:id - Delete student (Admin only)
- POST /:id/link-parent - Link parent to student (Admin only)
- DELETE /:id/unlink-parent/:parentId - Unlink parent from student (Admin only)
